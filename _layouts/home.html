How We Do It

We believe in some important modern web trends:

Single Page Applications - applications built using lots of JavaScript code on client side. User Interface is rendered and displayed without roundtrips to server. We communicate with server only for data - no HTML is rendered by server. We implement this architecture using AngularJS or IONIC Framework with JSON REST services. Benefits are obvious - applications behave like native desktop applications - UI is fast and responsive, users don't wait for operations.

Responsive Applications - when it's possible we prefer to build one application that adjusts it's look and behaviour to runtime platform, screen size and input interface. If mobile usage differs from desktop we build two separate web applications using different UI frameworks.

Cloud Services - when it's possible we use Cloud services for data storage and user management (Telerik Platform, Firebase, etc.) for server management simplification and lowering costs.

Node.js - for complicated and secure server logic we use Node.js based server applications - that gives us comfort of JavaScript everywhere, simplifies maintenance, lowers platform costs, empowers scalability and performance.

NoSQL Databases - we use modern document based databases (MongoDB, CouchDB) to simplify data storage and communication between tiers. NoSQL gives us freedom that accelerates development and simplifies changes. When necessary we use SQL Servers like Postgress and MySQL.
